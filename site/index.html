Как участвовать Философия

Путь простоты

Лучшее управление почти незаметно: статический сайт, открытая история изменений и понятная очередь модерации.

Качество без закрытости

Любой может предложить запись. Публикация требует источников, лицензии для фото и человеческого review.

Честная неопределённость

Спорные сведения получают статус uncertain, а не маскируются под окончательную истину.

Как знание попадает на сайт

  1. Участник отправляет форму, Issue или Pull Request.
  2. Автоматические проверки ищут дубли, отсутствие лицензий и ошибки схемы.
  3. Модератор задаёт вопросы или одобряет изменение.
  4. После merge сайт обновляется автоматически.

Техническое решение

На старте — обычные статические файлы. Данные живут в Git, медиа — во внешнем хранилище, обсуждения — в GitHub/форуме.

  • GitHub PR/Issues как очередь
  • GitHub Pages / Netlify / Vercel
  • Cloudinary/S3 для фотографий
  • CI для схемы и дублей

Агенты как садовые инструменты

Агенты помогают проверять данные, готовить описания, искать противоречия и ревьюить PR. Они не публикуют без людей и не выполняют опасные действия без подтверждения.
