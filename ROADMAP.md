# Roadmap

Roadmap фиксирует ближайшие шаги проекта без обещания сложной платформы. Каждый этап должен сохранять простоту поддержки.

## Этап 0 — основание

- [x] Описать назначение проекта.
- [x] Зафиксировать философию и принципы.
- [x] Добавить базовые правила участия и модерации.
- [x] Подготовить минимальный статический сайт.
- [x] Добавить локальную проверку структуры проекта.

## Этап 1 — первый полезный реестр

- [x] Уточнить поля `data/schema.yaml` на реальных примерах.
- [x] Добавить отдельную проверку записей на базовые поля и дубли.
- [x] Описать шаблон Issue для новой записи.
- [x] Описать шаблон Pull Request для изменения данных.
- [x] Подготовить минимальную форму подачи записи.
- [ ] Добавить несколько демонстрационных записей с источниками и лицензиями.
- [ ] Согласовать статусы уверенности и спорных записей на примерах.
- [ ] Собрать первую публичную витрину каталога из данных.

## Этап 2 — удобство участия

- [ ] Добавить инструкцию для авторов фотографий.
- [ ] Подготовить короткий путь для новичка: от наблюдения до предложенной записи.
- [ ] Настроить базовую публикацию статического сайта.
- [ ] Описать процесс регулярного модераторского обзора.
- [ ] Улучшить шаблон ответов модераторов и triage-подсказки.

## Этап 3 — качество и устойчивость

- [ ] Добавить поиск дублей по названию и синонимам.
- [ ] Ввести журнал спорных решений и повторных проверок.
- [ ] Подготовить публичный список источников и справочных материалов.
- [ ] Проверить доступность сайта и читаемость на мобильных устройствах.
- [ ] Оценить, нужна ли отдельная форма отправки материалов.

## Не цели ближайших этапов

- строить социальную сеть;
- хранить большие медиафайлы в Git;
- вводить сложную систему ролей до появления реальной нагрузки;
- заменять человеческую модерацию автоматическими решениями.
