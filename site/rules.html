Главная Вклад Правила Модерация Roadmap

Правила

Мягкий порядок, ясные границы

Правила должны быть короткими, понятными и применимыми без лишней бюрократии.

1. Точность

Не утверждаем больше, чем подтверждают источники и наблюдения.

2. Прозрачность

У любого фото есть автор, лицензия и источник. У любой спорной записи есть статус.

3. Уважение

Критика относится к записи, а не к человеку. Уточнение не равно нападению.

4. Мера

Мы не строим тяжёлую систему там, где достаточно GitHub и статического сайта.

Статусы записей

  • draft — черновик
  • uncertain — нужны уточнения
  • accepted — принято
  • deprecated — сохранено для истории
