# Agent workflow

AI-агенты помогают поддерживать сайт и реестр, но работают как помощники модераторов, а не как автономная власть.

## Роли агентов

### Explorer

- читает структуру проекта;
- находит дубли, большие файлы, устаревшие документы;
- не редактирует файлы.

### Planner

- превращает запрос в маленький план;
- отмечает риски и зависимости;
- предлагает, что можно сделать без инфраструктурного роста.

### Executor

- вносит локальные изменения;
- не делает push и не создаёт remote без подтверждения;
- не выполняет destructive actions без явного согласия.

### Reviewer

- проверяет ясность правил, согласованность документов, качество сайта;
- ищет противоречия и переусложнение.

### Verifier

- запускает `npm run check`;
- проверяет отсутствие `.crew/`, `node_modules/`, больших бинарников;
- подтверждает, что проект готов к PR.

## Безопасность

Агентам запрещено без явного подтверждения:

- создавать GitHub repo;
- делать `git push`;
- удалять историю;
- публиковать секреты;
- менять правила модерации без review;
- банить участников.

## Рекомендуемый цикл разработки

1. Человек формулирует цель.
2. Explorer/Planner уточняют масштаб.
3. Executor делает маленький PR.
4. Reviewer проверяет содержание и архитектуру.
5. Verifier запускает проверки.
6. Человек принимает решение.

## Главный критерий

Если агентная автоматизация создаёт больше работы, чем снимает, её следует удалить или упростить.
